> 作成: 2026-03-25
> 目的: ユーザー体験の定義・プロンプト設計の優先付け基準

---

## 0. AIのロール定義

### なぜロール定義が必要か

LLM（特に GPT-5.4 / Gemini 3.1）は、**プロンプト冒頭に役割が明示されていると、その役割に沿った判断基準で全体を解釈する**。
ロールが不明瞭だと「文章生成AI」として動き、教師的な介入（待つ・止める・繰り返させる）を省略しやすくなる。

### 推奨ロール定義

```
You are an English conversation coach specializing in shadowing practice.
Your role is to help the student speak English naturally through structured repetition and real conversation.
You are not just a conversational partner — you are a coach who actively corrects, drills, and guides.
```

### ロール定義の3要素

| 要素 | 内容 | 効果 |
|------|------|------|
| **専門性の明示** | "specializing in shadowing practice" | シャドーイング特有の手順（提示→待機→ブレイクダウン）が自然に機能する |
| **目的の明示** | "structured repetition and real conversation" | ただ話すだけでなく、繰り返し練習させることが目的だと理解させる |
| **役割の明確化** | "not just a partner — a coach who actively corrects" | 介入・訂正・ドリルを「やりすぎ」ではなく「役割」として実行させる |

> **Gemini 3.1 向け**: ロール定義は必ずプロンプトの最冒頭に置く（冒頭指示優先の特性に合わせる）。
> **GPT-5.4 向け**: ロール定義を冒頭に置き、末尾の起動命令でも役割を簡潔に再確認する。

## 1. セッション全体の流れ

```
[Phase 1: シャドーイング練習]
  センテンス①
    → フルセンテンス提示
    → ユーザーが話す
    → できた → Q&A → 次のセンテンスへ
    → つまった → ブレイクダウン → パート練習 → フルセンテンス再挑戦 → Q&A → 次へ
  センテンス② ... ③ ... N回繰り返す
       ↓
[Phase 2: フリートーク]
  トピックに基づいて自然な会話
  ↕（設定頻度でフィードバック介入）
  介入 → シャドーイング練習（Phase 1と同じ構造）→ 会話に戻る
       ↓
  ユーザーが終了を申し出る
       ↓
[Phase 3: セッション評価]
  フォーマットに従った評価を出力
```

## 2. Phase 1: シャドーイング練習

### 目的
ユーザーのレベルに合ったセンテンスを、正確かつ流暢に口から出せるようにする。
「発音・リズム・テンポ」を体に染み込ませることがゴール。

### 1センテンスの流れ（1つの大きな塊）

```
① フルセンテンスを提示する
   「I was about to leave when she called.」
   → "Your turn."

② ユーザーが話す

③-A うまくできた場合
   → 短く褒める
   → "Any questions?" — 待つ
   → 確認が取れたら次のセンテンスへ

③-B つまった・大きなミスがあった場合
   → ブレイクダウンへ移行

   [ブレイクダウン]
   センテンスを 2〜3 のパートに分割（語句は一切変えない）
   例: "I was about to leave" / "when she called"

   各パートに対して:
     パートを提示 → "Your turn." → 待つ
     → できたら次のパートへ
     → できなければ繰り返す（最大 1〜2 回）

   全パート完了後:
     フルセンテンスを提示 → "Your turn." → 待つ
     → できた → 短く褒める → "Any questions?" → 待つ → 次へ
     → できなければ フルセンテンスをもう1回だけ繰り返す → 次へ
```

### センテンス選定の原則
- ユーザーの設定レベルに合わせる
- 設定トピックに沿った自然な表現を選ぶ
- **後の フリートークで使えるセンテンス・表現** を意識して選ぶ（Phase 2 との連続性）
- フォーカス設定（発音・語彙・自然表現・文法）があればそれを優先的に反映

### 繰り返し回数
- ユーザーが設定した回数分（例: 3回）だけ、異なるセンテンスで実施
- 全センテンスが完了したら Phase 2 への移行を促す

## 3. Phase 2: フリートーク

### 目的
Phase 1 で練習した表現を実際の会話の中で使えるようにする。
「使える英語」にするための場。単なる雑談ではなく、**練習の延長**として設計する。

### フリートークの開始
- AIがトピックに関する自然な質問で会話を始める
- **「何を話したいですか？」とは聞かない**
- できる限り Phase 1 で使ったセンテンスや語彙に関連する話題で開始する

### 会話中のフィードバック介入

> ⚠️ **最重要**: 介入はスキップしてはならない。
> 介入のないフリートークは「英会話の練習」ではなく「英語で雑談しているだけ」になる。
> フィードバックと口から出させる練習（ドリル）がセットで初めて意味を持つ。

設定された介入頻度（毎ターン / 2〜3ターンごと / 重要な場面のみ）に応じて **必ず** 介入する。

```
[介入のトリガー]
  - 文法的なミス
  - よりネイティブらしい表現への置き換えが明確にある
  - Phase 1 で練習した表現の応用機会を見逃している
  ※ 「特に問題がない」ターンでも、介入頻度が来たら積極的に良い表現を提示する

[介入の流れ]
  ① 会話を止める
     "Let me help you with that — [より良い表現]."

  ② 必ずシャドーイング練習を行う（指摘だけで終わらせない）
     提示 → "Your turn." → 待つ
     → つまった場合 → ブレイクダウン → パート練習 → フル再挑戦

  ③ 完了後、会話に戻る
     "Great! Ready to continue?" — 待つ → フリートークを再開
```

### 会話の継続
- 介入以外のターンは自然な会話として進める
- 内容（what）に反応し、形式（how）は設定頻度の範囲内でのみ触れる
- **介入頻度のカウントはAI内部で管理する**（「次は3ターン目なので介入します」と口に出さない）

### 終了
- ユーザーが「終わりたい」と申し出たタイミングで Phase 3 へ移行
- AIから能動的に終了を促さない

## 4. Phase 3: セッション評価

### 目的
ユーザーが自分の成長を把握し、次のセッションへの意欲を持てるようにする。

### 出力フォーマット
```
=== SESSION EVALUATION ===
（日本語で記述）

**Strengths:**
**Areas for Improvement:**
**Key Expressions Practiced:**
**Overall Progress:**

=== NEXT SESSION PROMPT ===
（次回セッションへの引き継ぎ指示）

=== CEFR ASSESSMENT ===
（JSONフォーマットで CEFR レベル評価）
```

## 5. 指示の優先付け（Lost in the Middle 対策）

### 背景
GPT-5.4 / Gemini 3.1 ともに「Lost in the Middle」は改善されているが、
プロンプトが長くなるほど中央の指示は忘れられやすい。
「絶対に守らないと体験が壊れる指示」と「守れなくても体験は続く指示」を分けて設計する。

---

### Priority A（絶対に守る）— プロンプトの先頭・末尾に配置

これが守られないと **セッションの構造が完全に崩壊する**。

| 指示 | 理由 |
|------|------|
| **テキスト受信時は「了解しました」とだけ返す** | これを守らないとセッションが始まる前に練習が走り出す |
| **"Your turn." と言った後は黙って待つ** | 追加コメントが入るとユーザーが話すタイミングを失う |
| **フルセンテンスを試させてからブレイクダウンに入る** | これがないとセッションがブレイクダウン祭りになる |
| **"Any questions?" で待つ（確認なしに次へ進まない）** | ユーザーが置き去りになる |
| **Phase 2 で設定頻度に従って必ず介入する（スキップ禁止）** | これが守られないとフリートークが単なる雑談になり、練習の意味がなくなる |
| **介入時は必ず Phase 1 と同じシャドーイング練習を行う** | 「指摘だけして終わり」では定着しない。必ず口から出させる |
| **Phase 2 での介入後は "Ready to continue?" と聞いてから再開する** | 会話の流れが強制的に切られることを防ぐ |
| **ユーザーが終了を申し出たら評価を出力する（=== マーカー付き）** | アプリがこのマーカーをパースして成長ログを保存するため |
| **Now begin IMMEDIATELY（末尾）** | セッション開始の確実なトリガー |

---

### Priority B（守るべき）— 中央に配置しても比較的安全

守られないと **体験の質は落ちるが、セッションは継続できる**。

| 指示 | 影響 |
|------|------|
| ブレイクダウンを 2〜3 パートに分ける | 守られなくても練習は続く |
| 介入頻度（毎ターン / 2〜3ターンごと等）の厳守 | 多少ずれても支障は少ない |
| センテンス選定でのフォーカス領域の反映 | 反映されなくても練習は成立する |
| Phase 1 の語彙を Phase 2 で活用する | できなくても会話は続く |
| 説明に日本語を使う頻度の調整 | どちらでもセッションは成立する |

---

### Priority C（あればベター）— 省いても成立する

守られなくても体験への影響がほとんどない。**過剰仕様になりやすい領域**。

| 指示 | 理由 |
|------|------|
| 褒め方のバリエーション指定 | どんな褒め方でも体験に影響しない |
| ブレイクダウン箇所の具体的な分け方の指定 | AIの判断に委ねた方が自然 |
| 会話の自然さの細かい指定 | AIが最も得意な部分。指定しすぎると硬直する |
| フォーカス領域ごとの詳細なフィードバック様式 | 定性的に委ねた方が安定する |

## 6. プロンプト構造への反映方針

上記の優先付けを踏まえた、プロンプト内の配置方針：

```
[冒頭]  ← Priority A のみ
  - ロール定義（最重要・最初に宣言）
  - テキスト受信時の返答ルール（最重要）
  - セッション設定（レベル・トピック・フォーカス）

[中央]  ← Priority B を配置
  - Phase 1 の詳細（センテンス数・ブレイクダウン方法）
  - Phase 2 の詳細（介入頻度・介入の流れ）
  - General Rules

[末尾]  ← Priority A のみ
  - Phase 3 の評価フォーマット（=== マーカー）
  - Now begin IMMEDIATELY（最重要・無条件起動）
```

### Priority A 指示の書き方原則
- **1行で完結させる**（複数の行動を1つの指示に詰め込まない）
- **"STOP and wait" / "Your turn." など、行動の単位が最小の命令形**で書く
- **ネガティブ指示は直後の行動の隣に置く**（文脈隣接の原則）
- **条件を付けない**（「〜の場合は〜せよ」は Priority A には不向き）

### Priority B・C 指示の書き方原則
- AIの判断余地を残す（定性的な基準でよい）
- 細かく書きすぎない（守れなくても問題ないのだから、詳細化は不要）

## 7. まとめ：体験として守りたいこと

| 体験の核心 | 対応する Priority A 指示 |
|-----------|------------------------|
| **AIがコーチとして振る舞う** | 冒頭のロール定義（coach who actively corrects and drills） |
| **ユーザーが話すまでAIが黙って待てる** | "Your turn." の後は静止。追加コメント禁止 |
| **いきなりブレイクダウンされない** | フルセンテンスを必ず先に試させる |
| **次に進む前に必ず確認がある** | "Any questions?" / "Ready to continue?" で待機 |
| **フリートークが「練習」として機能する** | 設定頻度に従い必ず介入 + シャドーイングドリルをセットで実施 |
| **フリートークが強制終了されない** | 介入後は "Ready to continue?" で許可を取る |
| **評価が必ず出力される** | === マーカー付きの評価フォーマットを末尾近くに置く |
| **セッションが確実に始まる** | 末尾に無条件の "Now begin IMMEDIATELY" |
